Yii Presentation 2012

A high performance PHP5 framework best for developing Web 2.0 Applications

What?

  • MVC architecture
  • Purely OOP ( CComponent-based )
  • Promote DRY design
  • BSD License

History

  • Started by Qiang Xue in January 1, 2008
  • Yii 1.0 released December 3, 2008
  • Yii 1.1.0 release January 10, 2010

  • Yii 2.0 alpha verions released March 2012 (not guarantee)

    Yii team

  • Current Development team: 7 members

  • ~50 Translator (+15 languages)

    Priority

  • Code stability.

  • Code reliability.
  • Consistent style.

Why another framework? Reinventing the wheel?

  • Prado, the major source of ideas for Yii. (2004 nothing was out there)
  • Ruby on Rails, Convention over configuration, active record design pattern for its ORM layer. (...)

    has sensible defaults for almost all aspects of wiring your application. If you follow the prescribed conventions, you will write less code and spend less time developing your application.

  • Symfony, filter design and plug-in architecture
  • Joomla, modular design and message translation scheme
  • Java, C# (...)

I <3 features, do you?

  • DAO, Query Builder, Active Record, DB Migration

    Allows developers to model database data in terms of objects and avoid the tedium and complexity of writing repetitive SQL statements.

  • jQuery integration Integrated with jQuery, Yii comes with a set of AJAX-enabled widgets, such as auto-complete input field, treeview, data grid, which enables writing highly efficient and versatile user interface extremely simple.
  • RBAC, Authentication and authorization.
  • Internationalization (I18N) and localization (L10N) Yii supports message translation, date and time formatting, number formatting, and interface localization.
  • Layered caching scheme Yii supports data caching, page caching, fragment caching and dynamic content. The storage medium of caching can be changed easily without touching the application code.
  • Error handling and logging Errors are handled and presented more nicely, and log messages can be categorized, filtered and routed to different destinations.
  • Security Measures to help prevent your Web applications from attacks such as SQL injection, cross-site scripting (XSS), cross-site request forgery (CSRF), and cookie tampering.
  • Unit and functionality testing ( PPHUnit & Selenium )
  • Automatic code generation

++extra++ - Friendly with third-party code ( code from PEAR or Zend Framework ) - Form input and validation ( easy and safe, numerous helper methods and widgets to simplify the task for form input and validation ) - Skinning and Theming ( quickly switch the outlook of a Yii-power website )
